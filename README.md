# E-commerce Customer Churn Analysis using SQL

##  Project Overview
This project analyzes an e-commerce customer dataset to identify key factors that influence customer churn.  
The analysis is performed using MySQL to explore customer behavior, engagement patterns, and service-related attributes.

The objective is to help businesses understand why customers leave and how retention can be improved.

---

##  Database & Table Details
Database Name: ecomm  

The dataset contains the following key columns:

- CustomerID  
- Churn (0 = No, 1 = Yes)  
- Tenure  
- PreferredLoginDevice  
- CityTier  
- WarehouseToHome  
- PreferredPaymentMode  
- Gender  
- HourSpendOnApp  
- NumberOfDeviceRegistered  
- PreferredOrderCat  
- SatisfactionScore  
- MaritalStatus  
- NumberOfAddress  
- Complain  
- OrderAmountHikeFromlastYear  
- CouponUsed  
- OrderCount  
- DaySinceLastOrder  
- CashbackAmount  

---

##  Tools & Technologies Used
- MySQL  
- SQL Queries  
- Relational Database Design  

---

##  Key Analysis Performed
- Churn rate calculation  
- Churn by gender and marital status  
- Churn by preferred payment mode  
- Churn vs tenure analysis  
- Impact of complaints on churn  
- Churn by satisfaction score  
- Churn by preferred order category  
- Average cashback and churn relationship  

---

##  Business Insights
- Customers with low tenure show higher churn  
- Customers who raised complaints are more likely to churn  
- Low satisfaction scores are strongly associated with churn  
- Certain payment modes and order categories have higher churn risk  
- Higher engagement (orders, app usage) reduces churn probability
